# Descubrimiento de Arquitectura

## Objetivo
Identificar la arquitectura actual del proyecto y documentar sus componentes, patrones y decisiones arquitectónicas.

## Proceso
1. Analizar estructura de carpetas del proyecto
2. Identificar patrones arquitectónicos (MVC, Clean Architecture, Hexagonal, CQRS)
3. Detectar frameworks y librerías principales
4. Mapear dependencias entre módulos
5. Documentar decisiones arquitectónicas existentes
6. Identificar deuda técnica arquitectónica

## Artefactos generados
- Diagrama C4 de contexto y contenedores
- Mapa de dependencias entre módulos
- Lista de patrones arquitectónicos detectados
- Inventario de deuda técnica arquitectónica
- Recomendaciones de mejora

## Checklist
- [ ] ¿La estructura de carpetas refleja la arquitectura?
- [ ] ¿Hay separación clara de capas?
- [ ] ¿Las dependencias fluyen en una sola dirección?
- [ ] ¿Hay algún patrón anti-patrón presente?
- [ ] ¿La arquitectura soporta los requisitos no funcionales?
