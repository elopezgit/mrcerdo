# Análisis de Repositorio

## Objetivo
Analizar el repositorio del proyecto para entender su estructura, tecnologías, calidad de código y estado general.

## Dimensiones de análisis
1. **Estructura**: Organización de carpetas, módulos y proyectos
2. **Tecnologías**: Lenguajes, frameworks, librerías y herramientas detectadas
3. **Calidad**: Complejidad ciclomática, cobertura de pruebas, deuda técnica
4. **Dependencias**: Versiones de paquetes, vulnerabilidades conocidas
5. **Configuración**: Archivos de configuración, CI/CD, Docker, IaC
6. **Documentación**: README, documentación técnica, ADRs
7. **Colaboración**: Convenciones de commit, branching strategy, PRs

## Herramientas sugeridas
- Análisis estático: SonarQube, ESLint, Roslyn Analyzers
- Dependencias: Dependabot, Snyk, OWASP Dependency-Check
- Estructura: Tree CLI, cloc para conteo de líneas
- Git: GitStats, git shortlog para actividad

## Entregables
- Reporte de estructura del repositorio
- Inventario tecnológico (stack detectado)
- Métricas de calidad de código
- Lista de vulnerabilidades encontradas
- Recomendaciones de mejora priorizadas
